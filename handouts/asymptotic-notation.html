Asymptotic Notation:
\(O\), \(\Omega\), \(\Theta\) as Sets of Functions

CSS 343 - Data Structures, Algorithms & Discrete Mathematics II. Supplementary handout.
Course notes. Precise definitions of \(O\), \(\Omega\), \(\Theta\), \(o\), \(\omega\) as sets of functions, the relationships among them, how they apply to algorithms whose cost depends on the input, and exercises with solutions.

The single most clarifying idea in this topic is that each of \(O(g)\), \(\Omega(g)\), \(\Theta(g)\) is a set of functions. A function \(f\) either belongs to the set or it does not. That is a crisp true or false question, settled by exhibiting constants or ruling them out. Everything else follows from this.

1. What the notation describes

Asymptotic notation describes functions \(f:\mathbb{N}\to\mathbb{R}\), most often a running time or a memory footprint expressed as a function of the input size \(n\). We are interested in the order of growth of \(f\): how \(f\) scales as \(n\) becomes large, ignoring constant factors and the behavior at small \(n\). For large inputs the constant factors and lower order terms are dominated by the growth in \(n\) itself, so discarding them keeps the part of the cost that actually matters and makes the description independent of the machine and the implementation.

Standing assumption Every function placed inside this notation is assumed asymptotically nonnegative, meaning \(f(n)\ge 0\) for all sufficiently large \(n\). Without this assumption the sets defined below can be empty. Running times and memory sizes are nonnegative, so the assumption holds automatically in our setting.

2. The three definitions: \(\Theta\), \(O\), \(\Omega\)

Fix a reference function \(g\) (for example \(n\), \(n\lg n\), or \(n^2\)). Each notation below names a set of functions defined by an inequality that holds for all sufficiently large \(n\), up to constant factors.

Definition (Theta) \[ \Theta(g(n)) = \bigl\{\, f(n) : \exists\, c_1,c_2,n_0 \gt 0 \text{ such that } 0 \le c_1 g(n) \le f(n) \le c_2 g(n)\ \text{ for all } n \ge n_0 \,\bigr\}. \] \(\Theta\) gives a tight bound. A function \(f\) is in \(\Theta(g)\) when it is squeezed between two fixed multiples of \(g\) for large \(n\), so \(f\) equals \(g\) to within constant factors.
Definition (Big-O) \[ O(g(n)) = \bigl\{\, f(n) : \exists\, c,n_0 \gt 0 \text{ such that } 0 \le f(n) \le c\,g(n)\ \text{ for all } n \ge n_0 \,\bigr\}. \] \(O\) gives an upper bound: one constant, one side. It says \(f\) grows no faster than \(g\), to within a constant factor. It does not claim the bound is tight.
Definition (Big-Omega) \[ \Omega(g(n)) = \bigl\{\, f(n) : \exists\, c,n_0 \gt 0 \text{ such that } 0 \le c\,g(n) \le f(n)\ \text{ for all } n \ge n_0 \,\bigr\}. \] \(\Omega\) gives a lower bound: \(f\) grows at least as fast as \(g\), to within a constant factor.
Example (membership) \(\tfrac12 n^2 - 3n \in \Theta(n^2)\). We must produce \(c_1,c_2,n_0\) with \(c_1 n^2 \le \tfrac12 n^2 - 3n \le c_2 n^2\) for all \(n\ge n_0\). Dividing by \(n^2\) gives \(c_1 \le \tfrac12 - \tfrac3n \le c_2\). The right inequality holds for all \(n\ge 1\) with \(c_2 = \tfrac12\). The left inequality holds for all \(n\ge 7\) with \(c_1 = \tfrac{1}{14}\). So \(c_1=\tfrac1{14}\), \(c_2=\tfrac12\), \(n_0=7\) work. It is enough that some such constants exist.
Example (non-membership) \(6n^3 \notin O(n^2)\). Suppose for contradiction that there exist \(c_2,n_0 \gt 0\) with \(6n^3 \le c_2 n^2\) for all \(n\ge n_0\). Dividing by \(n^2\), it follows that \(n \le c_2/6\) for all \(n \ge n_0\). This is a contradiction, since \(c_2\) is constant while \(n\) grows without bound. Therefore \(6n^3\) is not in \(O(n^2)\), and in particular not in \(\Theta(n^2)\).

3. Membership versus equality: \(f \in O(g)\) or \(f = O(g)\)?

Since \(O(g)\) is a set of functions, the precise statement that \(g\) is an asymptotic upper bound for \(f\) is the membership \(f \in O(g)\). This is the form to prefer when you want to be unambiguous, and it is the form used throughout these notes.

The form \(f = O(g)\) is a widely used shorthand for the same membership. It is convenient, especially inside a larger formula, but it is a deliberate abuse of the equals sign and behaves differently from ordinary equality in two ways. First, it is read only from left to right: we write \(n = O(n^2)\), but we never write \(O(n^2) = n\). Second, it is not symmetric, so \(f = O(g)\) does not allow swapping the two sides. When asymptotic notation appears inside a formula, as in \(2n^2 + 3n + 1 = 2n^2 + \Theta(n)\), the \(\Theta(n)\) term stands for some function in the set \(\Theta(n)\), here the function \(3n+1\). The short answer to the question in the heading: \(f \in O(g)\) is the literally correct statement, and \(f = O(g)\) is accepted shorthand for it.

4. On the word “asymptotic”

In calculus an asymptote is a line that a curve approaches so that the gap between them tends to zero. The curve \(y = 1/x\) approaches the \(x\)-axis in this sense. One might therefore expect “asymptotic” here to mean that the difference between \(f\) and its bound tends to zero. That is not what these definitions say.

Reading of the term In this subject, “asymptotic” means only “for all sufficiently large \(n\),” that is, the behavior in the regime where \(n\) is large, always up to constant factors. The definitions control the ratio \(f(n)/g(n)\), keeping it between constants or above or below a constant. They say nothing about a vanishing difference. Read “asymptotically” as “once \(n\) is large enough,” not as “approaching arbitrarily closely.”

For instance, \(f(n) = 2n^2\) and \(g(n) = n^2\) satisfy \(f \in \Theta(g)\), yet the difference \(f(n) - g(n) = n^2\) grows without bound. The gap does not vanish. What stays controlled is the ratio, which is constantly equal to \(2\). The notation captures sameness of growth rate, not closeness of values.

5. Every polynomial of degree \(k\) is in \(\Theta(n^k)\)

Theorem (polynomials) Let \(p(n) = a_k n^k + a_{k-1}n^{k-1} + \cdots + a_1 n + a_0\) with \(a_k \gt 0\). Then \(p \in \Theta(n^k)\).
Proof. Let \(A = |a_0| + |a_1| + \cdots + |a_{k-1}|\) be the sum of the absolute values of the non-leading coefficients. If \(A = 0\) then \(p(n) = a_k n^k\) and the claim is immediate, so assume \(A \gt 0\).
Upper bound. For \(n \ge 1\) and each \(i \le k\) we have \(n^i \le n^k\). Therefore \[ p(n) \le |a_k|n^k + |a_{k-1}|n^{k-1} + \cdots + |a_0| \le \bigl(|a_k| + A\bigr)n^k = (a_k + A)\,n^k . \] Lower bound. For \(n \ge 1\), using \(n^i \le n^{k-1}\) for \(i \le k-1\), \[ p(n) = a_k n^k + \sum_{i=0}^{k-1} a_i n^i \;\ge\; a_k n^k - A\,n^{k-1} = n^{k-1}\bigl(a_k n - A\bigr). \] When \(n \ge 2A/a_k\) we have \(a_k n \ge 2A\), hence \(a_k n - A \ge \tfrac{a_k}{2}n\), and so \(p(n) \ge n^{k-1}\cdot \tfrac{a_k}{2}n = \tfrac{a_k}{2} n^k\).
Taking \(c_1 = \tfrac{a_k}{2}\), \(c_2 = a_k + A\), and \(n_0 = \max\!\bigl(1,\, 2A/a_k\bigr)\), we obtain \(0 \le c_1 n^k \le p(n) \le c_2 n^k\) for all \(n \ge n_0\). Therefore \(p \in \Theta(n^k)\). ∎

A constant function is a polynomial of degree \(0\), so any constant is in \(\Theta(n^0) = \Theta(1)\).

6. The theorem \(\Theta = O \cap \Omega\)

Theorem For every function \(g\), \(\;\Theta(g) = O(g) \cap \Omega(g)\). Equivalently, \(f \in \Theta(g)\) if and only if \(f \in O(g)\) and \(f \in \Omega(g)\).
Proof.
\(\Theta(g) \subseteq O(g)\cap\Omega(g)\). Let \(f \in \Theta(g)\). There exist positive constants \(c_1,c_2,n_0\) with \(0 \le c_1 g(n) \le f(n) \le c_2 g(n)\) for all \(n \ge n_0\). The inequality \(0 \le f(n) \le c_2 g(n)\) shows \(f \in O(g)\). The inequality \(0 \le c_1 g(n) \le f(n)\) shows \(f \in \Omega(g)\). Hence \(f \in O(g)\cap\Omega(g)\).
\(O(g)\cap\Omega(g) \subseteq \Theta(g)\). Let \(f \in O(g)\cap\Omega(g)\). Then there are \(c_2,n_2\) with \(f(n) \le c_2 g(n)\) for \(n \ge n_2\), and \(c_1,n_1\) with \(c_1 g(n) \le f(n)\) for \(n \ge n_1\). Put \(n_0 = \max(n_1,n_2)\). For all \(n \ge n_0\) both inequalities hold, so \(0 \le c_1 g(n) \le f(n) \le c_2 g(n)\). Hence \(f \in \Theta(g)\).
Each set is contained in the other, so they are equal. ∎

In particular, taking \(g(n) = n^k\), the theorem of §5 shows every degree-\(k\) polynomial lies in \(\Theta(n^k)\), and therefore in both \(O(n^k)\) and \(\Omega(n^k)\).

7. Best, average, and worst case

The running time of an algorithm is usually not determined by \(n\) alone. Two different inputs of the same size \(n\) can take different amounts of time. So “the running time” is not a single function of \(n\), and we cannot directly place it in \(O\), \(\Omega\), or \(\Theta\), which are defined for functions of \(n\). We resolve this by summarizing the spread of running times with three functions, each of which is a function of \(n\):

  • the worst-case time \(T_{\text{worst}}(n)\), the maximum running time over all inputs of size \(n\);
  • the best-case time \(T_{\text{best}}(n)\), the minimum running time over all inputs of size \(n\);
  • the average-case time \(T_{\text{avg}}(n)\), the mean running time over inputs of size \(n\) under a stated distribution.

Each of these is a function of \(n\), so each can be described with \(O\), \(\Omega\), or \(\Theta\). Choosing a case selects which function you describe; choosing one of \(O\), \(\Omega\), \(\Theta\) describes that function. The two choices are independent.

An example: a simple sort

Consider a sort such as insertion sort, or bubble sort with the standard early stop. On an input that is already in order it does a single linear pass and runs in \(\Theta(n)\) time. On an input in reverse order it does the maximum amount of work and runs in \(\Theta(n^2)\) time. So

quantity best case average case worst case
running time of the sort \(\Theta(n)\) \(\Theta(n^2)\) \(\Theta(n^2)\)

Can we say the sort “has a \(\Theta\) order”?

Only relative to a fixed case. The statement “the worst-case running time of the sort is \(\Theta(n^2)\)” is true and tight, because \(T_{\text{worst}}(n)\) really does grow like \(n^2\). The statement “the best-case running time is \(\Theta(n)\)” is true and tight as well. What we cannot do is attach a single \(\Theta\) to the unqualified running time, because across inputs of size \(n\) it ranges from order \(n\) to order \(n^2\); there is no single function that it always equals to within constant factors. The most we can say about the unqualified running time is that it is in \(O(n^2)\), since it is never more than a constant times \(n^2\), and in \(\Omega(n)\), since it is never less than a constant times \(n\). Both of these are as tight as possible.

Two careful statements

A \(\Theta(n^2)\) bound on the worst-case running time does not imply a \(\Theta(n^2)\) bound on the running time for every input. When the input is already sorted, the time is \(\Theta(n)\), not \(\Theta(n^2)\).

The unqualified running time therefore lies in both \(O(n^2)\) and \(\Omega(n)\), and these bounds are as tight as possible. The unqualified running time is not \(\Omega(n^2)\), because the already-sorted input runs in \(\Theta(n)\) time. Yet it is not contradictory to say that the worst-case running time is \(\Omega(n^2)\), because there is an input (the reversed one) that forces \(\Theta(n^2)\) time. The two statements concern two different functions: the running time on every input, and the worst-case running time \(T_{\text{worst}}(n)\).

8. \(o\) and \(\omega\): bounds that are not tight

An \(O\) bound may or may not be tight. The bound \(2n^2 \in O(n^2)\) is tight, while \(2n \in O(n^2)\) is not. To say that an upper bound is strictly loose, meaning \(f\) is of strictly smaller order than \(g\), we use little-oh. The definitions resemble those of \(O\) and \(\Omega\), but “for some constant \(c\)” is replaced by “for every constant \(c\).”

Definition (little-oh) \[ o(g(n)) = \bigl\{\, f(n) : \text{for every constant } c \gt 0,\ \exists\, n_0 \gt 0 \text{ such that } 0 \le f(n) \lt c\,g(n)\ \text{ for all } n \ge n_0 \,\bigr\}. \]
Definition (little-omega) \[ \omega(g(n)) = \bigl\{\, f(n) : \text{for every constant } c \gt 0,\ \exists\, n_0 \gt 0 \text{ such that } 0 \le c\,g(n) \lt f(n)\ \text{ for all } n \ge n_0 \,\bigr\}. \] Equivalently, \(f \in \omega(g)\) if and only if \(g \in o(f)\).

The contrast with \(O\) is exactly the quantifier on \(c\). In \(f \in O(g)\) the bound \(0 \le f(n) \le c\,g(n)\) holds for some constant \(c\). In \(f \in o(g)\) the bound \(0 \le f(n) \lt c\,g(n)\) holds for every constant \(c\). Intuitively, \(f \in o(g)\) means \(f\) becomes insignificant relative to \(g\) as \(n\) grows, and \(f \in \omega(g)\) means \(f\) becomes arbitrarily large relative to \(g\). These match the limits

\[ f \in o(g)\ \Rightarrow\ \lim_{n\to\infty}\frac{f(n)}{g(n)} = 0, \qquad\qquad f \in \omega(g)\ \Rightarrow\ \lim_{n\to\infty}\frac{f(n)}{g(n)} = \infty, \]

when the limit exists. Some authors take the first limit as the definition of \(o\). Examples: \(2n \in o(n^2)\) but \(2n^2 \notin o(n^2)\); and \(n^2/2 \in \omega(n)\) but \(n^2/2 \notin \omega(n^2)\).

9. Comparing functions

Many relational properties of real numbers carry over to asymptotic comparisons. Assume \(f\), \(g\), \(h\) are asymptotically positive.

  • Transitivity holds for all five relations. For example, \(f \in O(g)\) and \(g \in O(h)\) imply \(f \in O(h)\), and likewise for \(\Theta\), \(\Omega\), \(o\), \(\omega\).
  • Reflexivity holds for \(\Theta\), \(O\), \(\Omega\): \(f \in \Theta(f)\), \(f \in O(f)\), \(f \in \Omega(f)\). It fails for \(o\) and \(\omega\), since \(f \notin o(f)\).
  • Symmetry holds only for \(\Theta\): \(f \in \Theta(g)\) if and only if \(g \in \Theta(f)\).
  • Transpose symmetry links the upper and lower notations: \(f \in O(g)\) if and only if \(g \in \Omega(f)\), and \(f \in o(g)\) if and only if \(g \in \omega(f)\).

These properties suggest an analogy between comparing the growth of \(f\) and \(g\) and comparing two real numbers \(a\) and \(b\):

relation is like relation is like
\(f \in O(g)\) \(a \le b\) \(f \in o(g)\) \(a \lt b\)
\(f \in \Omega(g)\) \(a \ge b\) \(f \in \omega(g)\) \(a \gt b\)
\(f \in \Theta(g)\) \(a = b\)

We say \(f\) is asymptotically smaller than \(g\) when \(f \in o(g)\), and asymptotically larger when \(f \in \omega(g)\).

Where the analogy breaks One property of real numbers does not carry over: trichotomy. For real numbers \(a\) and \(b\), exactly one of \(a \lt b\), \(a = b\), \(a \gt b\) holds. For functions, two functions need not be comparable at all. It can happen that neither \(f \in O(g)\) nor \(f \in \Omega(g)\) holds. For example, \(n\) and \(n^{\,1+\sin n}\) cannot be compared, since the exponent \(1+\sin n\) oscillates between \(0\) and \(2\) and takes every value in between. Asymptotic order is a partial order, not a total one.

10. Measuring growth from data

An order of growth can also be estimated from measurements, not only established by proof. The central tool is the log-log plot. Measurement and analysis are complementary: measurement estimates the order and catches mistakes, while analysis proves the bound.

The key fact about log-log plots

Fact (power laws are lines on log-log) If \(T(n) = c\,n^b\) with \(c \gt 0\), then plotting \(\log T\) against \(\log n\) gives a straight line of slope \(b\) and intercept \(\log c\).
Proof. Take logarithms of \(T(n) = c\,n^b\): \(\;\log T(n) = \log c + b\,\log n\). Writing \(X = \log n\) and \(Y = \log T\), this is \(Y = (\log c) + b\,X\), the equation of a line with slope \(b\). ∎

So a straight line on log-log axes is the signature of a power law, and its slope is the exponent. To estimate the order of a power-law cost, measure it, plot it on log-log axes, and read the slope. The base of the logarithm does not matter, since changing base scales both axes by the same constant and leaves the slope unchanged.

How the prototype functions look

On linear axes only the fastest-growing function is visible; the slower ones are pressed against the axis and cannot be told apart, while the fast ones leave the frame. Linear axes are nearly useless for comparing growth. Two logarithmic rescalings each turn a different family into straight lines.

  • Semi-log (\(\log y\) against linear \(n\)): an exponential \(c\,a^n\) becomes a straight line, since \(\log y = \log c + (\log a)\,n\), with slope \(\log a\). Power laws appear concave down. A straight line on semi-log is the signature of exponential growth.
  • Log-log (\(\log y\) against \(\log n\)): a power law \(c\,n^b\) becomes a straight line of slope \(b\), by the fact above. Exponentials curve upward (convex). Logarithms flatten toward slope \(0\). The function \(n\log n\) is nearly a straight line of slope \(1\) that bends upward very slowly, since \(\log(n\log n) = \log n + \log\log n\).
function semi-log (\(\log y\), linear \(n\)) log-log
\(c\,a^{n}\) (exponential) straight line, slope \(\log a\) convex, curves upward
\(c\,n^{b}\) (power) concave down straight line, slope \(b\)
\(c\log n\) concave down concave down, flattening
\(n\log n\) concave down almost slope \(1\), slight upward bend

Measuring growth empirically: the doubling method

The standard procedure:

  1. Choose sizes on a geometric sequence, each double the previous. On a log axis these are evenly spaced.
  2. Measure a cost at each size. Prefer a deterministic operation count (comparisons, array writes) over wall-clock time, since counts carry no noise from caching, scheduling, or timer resolution. If you must use wall-clock time, repeat each size many times, take the median, and use sizes large enough that fixed overhead and timer granularity are negligible.
  3. Plot log-log and read the slope between two well-separated points: \(\;b \approx \dfrac{\log T_2 - \log T_1}{\log n_2 - \log n_1}\).

An equivalent and convenient shortcut is the doubling ratio. For \(T(n) = c\,n^b\),

\[ \frac{T(2n)}{T(n)} = \frac{c\,(2n)^b}{c\,n^b} = 2^b . \]

So the ratio of successive measurements at doubled sizes tends to \(2^b\): about \(2\) for linear, \(4\) for quadratic, \(8\) for cubic. Taking \(\log_2\) of the ratio gives the exponent directly, \(b = \log_2\!\bigl(T(2n)/T(n)\bigr)\). A ratio that keeps growing instead of settling signals a super-polynomial cost, such as exponential.

Choosing the axes and the range

  • Use log-log axes for a suspected power law, or semi-log for a suspected exponential. On linear axes an order of growth cannot be read.
  • Space the sizes geometrically so the points spread evenly across the log axis.
  • Use a wide range, several doublings, so the asymptotic slope is visible. At small \(n\) the constant factor and lower-order terms bend the curve, so read the slope from the large-\(n\) end where the leading term dominates.
  • Keep the measurement consistent (same machine, units, and build) so that only \(n\) varies.

Best and worst case in measurements: the special-case trap

The measured cost depends on the input, not on \(n\) alone (§7). If the best and worst cases differ in order, the slope you observe depends on which inputs you feed.

The figure shows real comparison counts for insertion sort on three families of inputs. Reversed inputs (the worst case) and random inputs (the average) both give slope near \(2\), revealing the \(n^2\) behavior. Sorted inputs (the best case) give slope near \(1\). If you timed only already-sorted arrays, the straight slope-\(1\) line would lead you to conclude that the sort runs in \(\Theta(n)\) time. The measurement is correct for that input family; the error is mistaking a special case for the general one. Other ways to land on a special case include a generator that produces nearly-sorted or all-equal data, sizes that are all powers of two for an algorithm sensitive to that, and inputs too small to be in the asymptotic regime.

How to avoid being misled

  • Decide which case you want, then construct inputs for it. For the worst case use known worst inputs (a reversed array for insertion sort). For the average draw many random inputs per size and report the median; for a worst-case sense, report the maximum over trials.
  • Use several trials per size and report the median, since a single timing can be an outlier.
  • Vary the input structure, not only its size, and check that the generator is not accidentally producing a special family.
  • Cross-check: compare the log-log slope against the doubling ratio, and an operation count against wall-clock time. Agreement is reassuring; disagreement points to noise or a special case.
  • Confirm the points actually lie on a line, and read the slope at large \(n\). A curve that bends means the cost is not a pure power law (it may be \(n\log n\), or two regimes, or sizes below the asymptotic range).
  • Remember the limit of the method: a straight log-log line over the tested sizes is evidence for a power law of that slope on that input family, not a proof of the order for all \(n\) or all inputs. Use measurement to estimate and to catch errors, and use analysis to establish the bound.

11. Exercises (with solutions)

  1. Let \(f\) and \(g\) be asymptotically nonnegative. Prove that \(\max(f(n),g(n)) \in \Theta(f(n)+g(n))\).
    solution For large \(n\) both functions are nonnegative. Then \(\max(f,g) \le f+g\), giving the upper bound with \(c_2 = 1\). Also \(f+g \le 2\max(f,g)\), so \(\max(f,g) \ge \tfrac12(f+g)\), giving the lower bound with \(c_1 = \tfrac12\). Hence \(\tfrac12(f+g) \le \max(f,g) \le f+g\), so \(\max(f,g) \in \Theta(f+g)\).
  2. Show that for any real constants \(a\) and \(b\) with \(b \gt 0\), \(\;(n+a)^b \in \Theta(n^b)\).
    solution For \(n \ge 2|a|\) we have \(|a| \le n/2\), hence \(\tfrac{n}{2} \le n+a \le \tfrac{3n}{2} \le 2n\), and \(n+a \gt 0\). Raising to the power \(b \gt 0\) preserves the inequalities: \(\bigl(\tfrac{n}{2}\bigr)^b \le (n+a)^b \le (2n)^b\), that is \(2^{-b} n^b \le (n+a)^b \le 2^{b} n^b\). So \(c_1 = 2^{-b}\), \(c_2 = 2^{b}\), \(n_0 = 2|a|\) (and at least \(1\)) work, and \((n+a)^b \in \Theta(n^b)\).
  3. Explain why “the running time of algorithm \(A\) is at least \(O(n^2)\)” is meaningless.
    solution \(O(n^2)\) is a set of upper bounds. The phrase “at least” asks for a lower bound, but combining it with an upper-bound symbol imposes no constraint: saying a quantity is “at least” bounded above by something rules nothing out. A lower bound is expressed with \(\Omega\), as in “the running time is \(\Omega(n^2)\).”
  4. Is \(2^{\,n+1} \in O(2^n)\)? Is \(2^{\,2n} \in O(2^n)\)?
    solution Yes for the first: \(2^{\,n+1} = 2\cdot 2^n \le 2\cdot 2^n\), so \(c = 2\), \(n_0 = 1\) work. No for the second: \(2^{\,2n} = (2^n)^2\). If \(2^{\,2n} \le c\,2^n\) held for all \(n \ge n_0\), then dividing by \(2^n\) gives \(2^n \le c\) for all \(n \ge n_0\), a contradiction since \(c\) is constant. So \(2^{\,2n} \notin O(2^n)\); in fact \(2^{\,2n} \in \omega(2^n)\).
  5. Prove \(\Theta(g) = O(g) \cap \Omega(g)\).
    solution This is the theorem of §6; its proof is given there. The upper half of the \(\Theta\) sandwich is exactly the \(O\) condition, the lower half is exactly the \(\Omega\) condition, and taking the maximum of the two thresholds \(n_0\) reassembles the sandwich.
  6. Prove that the running time of an algorithm is \(\Theta(g(n))\) if and only if its worst-case running time is \(O(g(n))\) and its best-case running time is \(\Omega(g(n))\).
    solution Here “the running time is \(\Theta(g)\)” means that for every input of size \(n\) the time lies between \(c_1 g(n)\) and \(c_2 g(n)\) for large \(n\). (If.) Suppose the worst-case time satisfies \(T_{\text{worst}}(n) \le c_2 g(n)\) and the best-case time satisfies \(T_{\text{best}}(n) \ge c_1 g(n)\) for large \(n\). Any input of size \(n\) takes time between \(T_{\text{best}}(n)\) and \(T_{\text{worst}}(n)\), hence between \(c_1 g(n)\) and \(c_2 g(n)\). So the running time is \(\Theta(g)\). (Only if.) If every input runs in time within \([c_1 g(n), c_2 g(n)]\), then the maximum over inputs satisfies \(T_{\text{worst}}(n) \le c_2 g(n)\), so the worst case is \(O(g)\), and the minimum satisfies \(T_{\text{best}}(n) \ge c_1 g(n)\), so the best case is \(\Omega(g)\).
  7. Prove that \(o(g) \cap \omega(g)\) is the empty set.
    solution Suppose \(f \in o(g) \cap \omega(g)\). From \(f \in o(g)\), applying the definition with \(c = 1\), there is \(n_1\) with \(f(n) \lt g(n)\) for all \(n \ge n_1\). From \(f \in \omega(g)\), applying the definition with \(c = 1\), there is \(n_2\) with \(g(n) \lt f(n)\) for all \(n \ge n_2\). For \(n \ge \max(n_1,n_2)\) both \(f(n) \lt g(n)\) and \(g(n) \lt f(n)\) hold, a contradiction. Hence the intersection is empty.
  8. Extend the notation to two parameters \(n\) and \(m\) that grow independently. Given \(g(n,m)\), define \[ O(g(n,m)) = \bigl\{ f(n,m) : \exists\, c,n_0,m_0 \gt 0 \text{ such that } 0 \le f(n,m) \le c\,g(n,m) \text{ for all } n \ge n_0 \text{ or } m \ge m_0 \bigr\}. \] Give corresponding definitions for \(\Omega(g(n,m))\) and \(\Theta(g(n,m))\).
    solution \[ \Omega(g(n,m)) = \bigl\{ f(n,m) : \exists\, c,n_0,m_0 \gt 0 \text{ such that } 0 \le c\,g(n,m) \le f(n,m) \text{ for all } n \ge n_0 \text{ or } m \ge m_0 \bigr\}, \] \[ \Theta(g(n,m)) = \bigl\{ f(n,m) : \exists\, c_1,c_2,n_0,m_0 \gt 0 \text{ such that } 0 \le c_1 g(n,m) \le f(n,m) \le c_2 g(n,m) \text{ for all } n \ge n_0 \text{ or } m \ge m_0 \bigr\}. \] Each keeps the one-parameter shape, with the condition required once \(n \ge n_0\) or \(m \ge m_0\).
  9. You run the doubling experiment and obtain successive ratios \(T(2n)/T(n)\). What order of growth does each sequence suggest? (a) \(2.05,\ 2.02,\ 2.01\); (b) \(3.9,\ 3.95,\ 3.98\); (c) \(2.4,\ 2.3,\ 2.2,\ 2.1\) (above \(2\), slowly decreasing toward it).
    solution (a) The ratio tends to \(2 = 2^1\), so \(b = 1\): linear, \(\Theta(n)\). (b) The ratio tends to \(4 = 2^2\), so \(b = 2\): quadratic, \(\Theta(n^2)\). (c) A ratio that sits just above \(2\) and decreases toward it is the signature of \(\Theta(n\log n)\): for \(T = n\log_2 n\), \(T(2n)/T(n) = 2\bigl(1 + 1/\log_2 n\bigr)\), which exceeds \(2\) and decreases toward \(2\) as \(n\) grows.
  10. You time a sort and the log-log plot of time against \(n\) is a straight line of slope \(1\) over the sizes you tried. Give two different explanations, and describe one further experiment that distinguishes them.
    solution Either (i) the algorithm genuinely runs in \(\Theta(n)\) time, or (ii) the algorithm is slower in general (for example \(\Theta(n^2)\) in the worst case) but you fed it best-case inputs, such as already-sorted arrays, which run in \(\Theta(n)\). To distinguish, rerun on a different input family of the same sizes, in particular reversed or random inputs. If the slope stays near \(1\), the linear behavior is genuine; if it rises to about \(2\), you were measuring the best case.
