ICA 03 — Tree Recursion

CSS 343 · Session 3 (Trees & Binary Search Trees)
At a glance
Format: in-class, individual Language: C++17 Starter: ica03.cpp Submit: your finished ica03.cpp

Every operation on a tree is the same move: do something with the node, then recurse on its subtrees. This activity drills that pattern four ways. Start from ica03.cpp — it already defines the node, builds a test tree, and prints your results. You only fill in the function bodies marked TODO.

Check yourself Hand-draw the tree that buildExample() builds (insert 50 30 70 20 40 60 80 10 45 into an empty BST), then compare your program's numbers against your drawing.

Tasks

Implement each function using recursion. Do not change buildExample(), main(), or the provided helpers.

  1. int height(Node* t) — the height of the tree. An empty tree has height -1; a single node has height 0.
  2. int countLeaves(Node* t) — how many leaves (nodes with no children) the tree has.
  3. int countOneChild(Node* t) — how many nodes have exactly one child.
  4. bool contains(Node* t, int key) — whether key is in the tree. Use the BST ordering so you follow only one root-to-leaf path (don't scan the whole tree).

Stretch (optional)

  • bool isValidBST(Node* t) — is the tree a valid BST? For every node, its key must be greater than every key in its left subtree and less than every key in its right subtree.

Extra credit — expression trees

The starter also defines an expression tree: each leaf holds an operand (val), each internal node holds a binary operator (op ∈ + − * /). Implement:

  • int evalExprTree(ExprNode* t) — evaluate the expression. A leaf is its val; an internal node applies its op to the values of its two subtrees.

main runs your one function on several trees of increasing complexity — from (2 + 3) * 4 up to 5 * ((3 * 4) + (6 / 2) - (8 + 2)). One correct recursion handles them all; hand-check the small ones.

Build & run

g++ -std=c++17 ica03.cpp -o ica03 && ./ica03

Work on the CSS Linux lab (see the Computing Environment handout). Your code uses new to build the tree — memory leaks are not checked on this activity, but ICA 04 will be the first leak-checked one, so it's worth skimming the Valgrind handout now.

Submit

Your finished ica03.cpp (with all four TODOs implemented; stretch and extra credit optional).
