CSS 343 · Session 4 (Balanced Trees I: AVL)
At a glance
Format: in-class, individual Language: C++17 Starter: ica04.cpp Submit: your finished ica04.cpp ⚠ First leak-graded ICA

You build the AVL machinery from the lecture: balance factors, the two rotations, the four-case rebalance, and insert — then feed it the input that destroys a plain BST (a sorted stream) and watch the height stay logarithmic. Start from ica04.cpp: the node (with its height field), the helpers, and a unit-test battery in main are given. You only fill in the TODO bodies.

Tasks

  1. bf / fix — balance factor; recompute a node's stored height.
  2. rotateRight / rotateLeft — O(1) pointer surgery; fix heights, child first.
  3. rebalance — the four-case dispatch (LL / RR / LR / RL — the double cases convert then rotate).
  4. insert — plain BST insert (ignore duplicates), then rebalance on the way back up.
  5. isAVL — |bf| ≤ 1 at every node · contains — BST search.

Stretch (+1 extra credit): erase with rebalancing on the way up (successor promotion for the two-child case — and don't leak the removed node).

The tests run themselves

Build and run — main reports each check as you go, so fill the TODOs top-to-bottom and re-run:

g++ -std=c++17 -g ica04.cpp -o ica04 && ./ica04
  • T1 basics — empty height, single node, contains hit/miss
  • T2–T5 one test per rotation case (RR · LL · LR · RL) — 3 keys each, middle key must end at the root
  • T6 duplicates are ignored
  • T7 a shuffled batch — sorted in-order, balanced, height bound
  • T8 the application: a ticket registry hands out sorted ids 1..1023 — the plain-BST worst case; your AVL must stay ≤ 1.44·log₂N ≈ 14 (a correct AVL lands at 9)
  • T9 the mirror worst case, reverse-sorted
Memory is graded — first time Every new must be freed. When all tests pass, run valgrind --leak-check=full ./ica04 on the CSS Linux lab — grading requires “All heap blocks were freed” and “ERROR SUMMARY: 0 errors.” See the Valgrind and Computing Environment handouts.
