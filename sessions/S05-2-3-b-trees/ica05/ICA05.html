You build the split-and-promote machinery from the lecture. The Node struct, the recursive insert driver, search, and a unit-test battery are given — you implement the two primitives the driver leans on, then feed the tree the sorted stream that destroys a plain BST and watch every leaf stay at the same depth.

Tasks

  1. addKey — insert a key (and the child to its right) into a node, keeping key[] sorted and the children aligned.
  2. split — turn a temporary 4-node into two 2-nodes, promoting the middle key via the out-parameter.

The tests run themselves

Build and run — main reports each check as you go, so fill the TODOs top-to-bottom and re-run:

g++ -std=c++17 -g ica05.cpp -o ica05 && ./ica05
  • T1–T2 basics — a single 2-node; a 2-node grows to a 3-node (no split)
  • T3 the first split — the root grows up
  • T4 a cascading split from below
  • T5 duplicates are ignored
  • T6 a shuffled 17-key batch — valid 2-3, sorted, every leaf at the same depth
  • T7 the application: sorted 1..1000 — a plain BST would be a 999-deep path; your 2-3 tree must stay ≤ log₂ N deep
  • T8 the mirror worst case, reverse-sorted
Memory is graded When all tests pass, run valgrind --leak-check=full ./ica05 on the CSS Linux lab — grading requires “All heap blocks were freed” and “ERROR SUMMARY: 0 errors.” The 2-3 nodes are real heap allocations — every split must leave nothing stranded.
