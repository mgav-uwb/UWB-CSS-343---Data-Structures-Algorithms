At a glance
Format: in-class, individual Language: C++17 Starter: ica06.cpp Submit: your finished ica06.cpp ⚠ Leak-graded

You implement the two reheapify walks from the lecture — swim and sink — plus the Θ(n) bottom-up heapify. The array-backed MaxHeap, insert, and delMax are given on top of your primitives, and the battery finishes by heapsorting 2047 keys.

Tasks

  1. swim — carry an oversized key up: while it beats its parent (k/2), swap.
  2. sink — push an undersized key down: swap with the larger child while that child beats it.
  3. heapify — load a raw array, then sink every internal node from n/2 down to 1 (bottom-up, Θ(n)).

The tests run themselves

Build and run — main reports each check as you go, so fill the TODOs top-to-bottom and re-run:

g++ -std=c++17 -g ica06.cpp -o ica06 && ./ica06
  • T1 insert (swim) keeps the heap invariant
  • T2 delMax (sink) returns items largest-first
  • T3 heapify a shuffled array — valid heap, same contents
  • T4 heapify vs repeated insert — same heap contents
  • T5 heapify ascending input (the max-heap worst case)
  • T6 the application: heapsort 1..2047 — heapify + drain, out in sorted order
Memory is graded When all tests pass, run valgrind --leak-check=full ./ica06 on the CSS Linux lab — grading requires “All heap blocks were freed” and “ERROR SUMMARY: 0 errors.” The heap lives in a vector — a clean run also catches out-of-bounds indexing.
