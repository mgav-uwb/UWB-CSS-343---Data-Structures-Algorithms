You implement the probe loops from the lecture: insert (probe forward, stop on a duplicate, place in the first empty slot, resize at load factor ½) and search (probe until the key or an empty slot). hashCode and resize are given.

Tasks

  1. insert — home slot via hashCode; linear-probe forward with wraparound; no duplicates; bump n; call resize when 2·n ≥ M.
  2. search — probe forward from the home slot; true on the key, false at the first EMPTY (the chain ends there).

The tests run themselves

Build and run — main reports each check as you go, so fill the TODOs top-to-bottom and re-run:

g++ -std=c++17 -g ica07.cpp -o ica07 && ./ica07
  • T1 basics — insert then search, hit and miss
  • T2 collisions — keys sharing a home slot land in distinct slots
  • T3 duplicates — reinserting a key doesn't grow n
  • T4 resize — crossing load factor 0.5 doubles M and preserves the keys
  • T5 search after resize — early keys survive a later rehash
  • T6 the application: 1000 keys through repeated resizes, membership verified
Memory is graded When all tests pass, run valgrind --leak-check=full ./ica07 on the CSS Linux lab — grading requires “All heap blocks were freed” and “ERROR SUMMARY: 0 errors.” The table lives in a vector — a clean run also catches out-of-bounds probing.
