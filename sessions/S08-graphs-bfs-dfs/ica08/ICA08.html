CSS 343 · Session 8 (Graphs I: Representations, BFS/DFS, Topological Sort)
At a glance
Format: in-class, individual Language: C++17 Starter: ica08.cpp Submit: your finished ica08.cpp ⚠ Leak-graded

You implement the lecture's two traversals on a directed adjacency list — the DFS recursion and BFS with a queue — plus Kahn's topological sort as an optional stretch. The Graph struct and the DFS wrapper are given; every test graph sorts its adjacency lists so your output order is deterministic.

Tasks

  1. dfsVisit — mark, record, recurse into each unvisited neighbor.
  2. bfs — queue-driven; mark a vertex seen when you enqueue it (not when you dequeue).
  3. topoSort — stretch (extra credit; do it if you finish early): Kahn's algorithm — in-degrees, a queue of zero-in-degree vertices, pop-append-decrement. A cycle leaves the output short.

The tests run themselves

Build and run — main reports each check as you go, so fill the TODOs top-to-bottom and re-run:

g++ -std=c++17 -g ica08.cpp -o ica08 && ./ica08
  • T1–T2 DFS — reaches everything reachable, in depth-first order
  • T3 BFS — visits by layer (fewest edges first)
  • T4 a valid topological order (stretch)
  • T5 a cycle has no topological order (short output — stretch)
  • T6 a second graph — a chain — sanity-checks everything you wrote

T4, T5, and T6's topo check report [FAIL] until you attempt the stretch — that costs you nothing; the core is DFS + BFS.

Memory is graded When all tests pass, run valgrind --leak-check=full ./ica08 on the CSS Linux lab — grading requires “All heap blocks were freed” and “ERROR SUMMARY: 0 errors.” Vectors and a queue only — a clean run also catches bad vertex indexing.
