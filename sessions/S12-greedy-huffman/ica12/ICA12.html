At a glance

You build the Huffman coding trie with the greedy merge — repeatedly join the two least-frequent nodes via the given min-priority-queue — then walk it to extract each symbol's prefix-free code. The starter gives the data structures, helpers, and a self-running test battery; you fill the TODOs.

Tasks

  1. buildHuffman — load the leaves into the min-PQ; repeatedly pop the two smallest, join them under a new internal node, push it back; return the last node standing.
  2. extractCodes — recursive walk: left appends '0', right appends '1'; record the path at each leaf.

The tests run themselves

g++ -std=c++17 -g ica12.cpp -o ica12 && ./ica12
  • T1–T2 a valid root; every symbol gets a code
  • T3 the codes are prefix-free
  • T4 the application: the total weighted code length is optimal
  • T5–T8 monotonicity, skewed frequencies, two symbols, decode-of-encode round-trip
Memory is graded When all tests pass, run valgrind --leak-check=full ./ica12 on the CSS lab — grading requires “All heap blocks were freed” and “ERROR SUMMARY: 0 errors.” The trie is real heap allocation — the given destroy must reach every node you create. See the Valgrind handout.
