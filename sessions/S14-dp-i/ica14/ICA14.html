You turn exponential recursion into linear DP: fibMemo (top-down) and fibTab (bottom-up), then the two classics — rod cutting (1-D) and longest common subsequence (2-D). The starter gives the data structures, helpers, and a self-running test battery; you fill the TODOs.

Tasks

  1. fibMemo — recursion + a cache: answer each subproblem once.
  2. fibTab — the same table, filled bottom-up with a loop.
  3. rodCut — best[len] = max over first-cut choices of price[cut] + best[len−cut].
  4. lcs — the (m+1)×(n+1) prefix table: diagonal +1 on a match, else max(up, left).

The tests run themselves

g++ -std=c++17 -g ica14.cpp -o ica14 && ./ica14
  • T1–T2 fibMemo/fibTab agree — and fib(90) only terminates if the cache works
  • T3, T6 rodCut — the CLRS price table + edge cases
  • T4–T5 lcs — classic instances + edge cases
