At a glance
Format: in-class, individual Language: C++17 Starter: ica17.cpp Submit: your finished ica17.cpp ⚠ Leak-graded

You implement the two halves of Kleene's construction from the lecture: epsClosure (everything reachable through ε-edges) and simulate — run the whole SET of possible states across the text, one character at a time. The starter gives the data structures, helpers, and a self-running test battery; you fill the TODOs.

Tasks

  1. epsClosure — DFS/BFS over the ε-digraph; the input states themselves are included.
  2. simulate — start from the closure of {0}; for each text character, advance every state that can consume it, then close again; accept iff the accept state is in the final set.

The tests run themselves

g++ -std=c++17 -g ica17.cpp -o ica17 && ./ica17
  • T1 epsClosure on A*B's machine
  • T2–T4 simulate A*B — accepts B/AB/AAAB, rejects A/BA/AABB and the empty string
  • T5 a second machine, (A*B|AC)D
Memory is graded When all tests pass, run valgrind --leak-check=full ./ica17 on the CSS lab — grading requires “All heap blocks were freed” and “ERROR SUMMARY: 0 errors.” Sets and vectors only — a clean run still catches bad state indexing. See the Valgrind handout.
