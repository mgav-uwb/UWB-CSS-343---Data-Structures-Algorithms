Course information

This core computer science course covers the internal workings of algorithms and data structures, from mathematical principles to implementation in C++. We begin with analysis of algorithms — how to measure an algorithm's time and memory cost and reason about it formally — and apply that lens to abstract data types (binary search trees, balanced trees, heaps and priority queues, hash tables, graphs, tries), algorithm-design paradigms (greedy, divide-and-conquer, dynamic programming), the discrete mathematics underpinning them (induction, counting, recurrences, graph theory), and a concluding unit on regular expressions and finite automata. Object-oriented design and design patterns are exercised in the group programming project.

The objective is to refine and extend the concepts and practical skills introduced in CSS 342.

The subject matter is highly technical, and summer is compressed — expect to commit 12+ hours per week outside of class and to stay focused the entire term.

Textbooks

The primary "text" for this course is its own lecture materials — the slides, live demos, notes, and the worked examples developed in class. The following are free, openly licensed references; nothing must be purchased.

Primary reference (free online):

  • Algorithms, 4th ed. — Robert Sedgewick & Kevin Wayne (Princeton). Read it on the free booksite, algs4.cs.princeton.edu — it carries the section text, all the code, exercises, and data; nothing to purchase. We follow it across the course: analysis (§1.4), binary search trees (§3.2), balanced trees (§3.3), sorting (§2), hashing (§3.4), graphs (§4), and strings/regex/compression (§5). The booksite code is Java; the lecture materials provide C++ equivalents for every assigned section, so you read the ideas without the language barrier.

Secondary references (free online):

  • Open Data Structures (in C++) — Pat Morin. opendatastructures.org — a C++ reference for the core data structures, handy when you want the code in our language.
  • Algorithms — Jeff Erickson (University of Illinois). jeffe.cs.illinois.edu/teaching/algorithms — our reference for the topics Sedgewick is light on: dynamic programming, greedy correctness, recurrences, and the sorting lower bound.

Discrete mathematics (free references):

  • Charles A. Cusack & David A. Santos, An Active Introduction to Discrete Mathematics and Algorithms (2018).
  • William Chen, Discrete Mathematics.

Optional, for the curious (not required):

  • Cormen, Leiserson, Rivest & Stein (CLRS), Introduction to Algorithms — the encyclopedic algorithms reference.
  • Skiena, The Algorithm Design Manual.
  • Graham, Knuth & Patashnik, Concrete Mathematics.
  • Stroustrup, Programming: Principles and Practice Using C++; Meyers, Effective C++ — for C++ depth.

How the course runs

Each session is a 2.5-hour block = lecture + an in-class activity. Lectures are interactive — questions are encouraged; raise your hand. Because much of the material is diagrams and live code, taking notes by hand is often easiest.

  • Programming assignments (PAs). Four C++ builds, graded on a Linux machine with g++ (see the Computing Environment page). If your code does not compile, you receive no correctness points — state any required flags (e.g. -std=c++17) in your submission. PAs are done individually except the final group project. Submit a zip of source on Canvas plus, where required, a separate report (PDF/Word). See the grading rubric and coding-standards pages.
  • In-class activities (ICAs). Each topic session ends with a short implement-the-algorithm activity begun in class and finished at home if needed (this is the course's homework — there is no separate lab). ICAs require attendance; an access code is given in class.
  • Quizzes. Three kinds: a reading quiz before each topic (multiple attempts, due before the module — take it early); a module quiz after each topic (one attempt, in class); and a programming quiz after each programming assignment (one attempt, timed, in class) that checks you understand your own submission — the spec, your implementation of specific functions, and your design choices.
  • Exams. A midterm (Session 10, Thu Jul 23) covering Sessions 1–9, and a comprehensive final (Session 18, Thu Aug 20) weighted more heavily than the midterm. No make-up exams, earlier or later.
  • Participation. In-class activities and discussion. If you are not in class, you cannot participate.

See the course Schedule page for the dated topic-by-topic plan.

Policies

Academic integrity. Zero tolerance for plagiarism and cheating; all work is checked. If misconduct is suspected: (1) the instructor contacts you to discuss; (2) refusal to engage is reported to the school with documentation; (3) if you are responsible and admit it before referral, you receive zero on the work but are not reported; (4) if responsible and you do not admit it, the case is referred to the school, which may result in failing the course with a recorded violation. Programming assignments are individual (except the group project) — collaboration or copied code is misconduct.

Late work. Submit on Canvas (not by email). Late assignments are not accepted, under any circumstances — plan around the compressed summer schedule and submit early.

Generative AI. AI tools (ChatGPT, Copilot, etc.) may be used for learning and for limited assistance if disclosed: cite the tool (as a website URL) in your references, summarize your prompts, and add a comment/README noting any AI-assisted code. For whole programming assignments, quizzes, and exams, AI is prohibited — you may get help on a few lines, but copying a whole solution defeats the purpose of the course and will be treated as misconduct. Example disclosure: "AI usage: ~20 lines in function merge were drafted with Copilot; prompt: '…'." If you used no AI on a piece of work, state: "I did not use any AI tools for this assignment. I assert that all work submitted is entirely my own."

Attendance. No lecture recordings and no make-up lectures. In-class exercises and quizzes are not always pre-announced; missing class can mean missing them. If you must miss for illness, family emergency, or unavoidable work obligations, you may make up in-class work within a week with documentation — but you remain responsible for all material covered. Come on time, prepared (take the reading quiz first), and ready to participate; no web surfing, messaging, or phone calls during class.

Disability accommodations (DRS). To request accommodations, contact Disability Resources for Students early (within the first 1–2 weeks). Have your DRS counselor contact the instructor. Approved exam accommodations must be requested through DRS at least 10 business days before each exam.

Grade questions. Bring any grading concern to the instructor (not the grader) within one week of the work being returned; after that, grades are final.
